S1
SRS Writer — Written Sections
Deliverable: SRS document sections 1–4 · Source: README.md · DATABASE.md · TEAMMATES.md
1. Introduction
1.1 Purpose
State that this SRS defines requirements for SmartRent
1.2 Scope
What the platform does (browse, visit, apply, manage) and does not do (payment, chat)
1.3 Definitions
JWT, Feign, RBAC, Microservice, DTO, ORM, Eureka, Gateway
1.4 References
Spring Boot docs, project README.md, DATABASE.md
2. Overall Description
2.1 Perspective
4 microservices, 4 isolated SQL Server DBs, React frontend, Docker
2.2 User classes
Tenant (browse/apply/review), Landlord (list/approve), Admin (moderate)
2.3 Environment
Java 21, Spring Boot 3.2.5, SQL Server 2022, Docker Compose
2.4 Constraints
JWT stateless auth, no cross-DB SQL joins, services communicate via Feign only
2.5 Assumptions
Single region deployment, English language, Docker available
3. Functional Requirements
FR-01 – FR-06
Authentication: register, login, JWT generation, role assignment, status checks
FR-07 – FR-15
Property Management: CRUD, image upload, search/filter, admin approval
FR-16 – FR-22
Visit Management: create, approve, reject, reschedule, cancel, conflict check
FR-23 – FR-31
Rental Applications: create, upload docs, approve, reject, withdraw
FR-32 – FR-36
Reviews & Favorites: CRUD reviews, add/remove favorites, one review per tenant
FR-37 – FR-42
Admin: list users, update status, platform stats, approve properties/landlords
4. Non-Functional Requirements
4.1 Security
JWT expiry 24h, BCrypt password hashing, HTTPS in production
4.2 Performance
Property search results returned in < 2 seconds
4.3 Scalability
Each microservice independently deployable and scalable
4.4 Availability
Docker health checks, Eureka re-registration on restart
4.5 Maintainability
AOP logging, Swagger/OpenAPI docs, MapStruct mappers

Number every functional requirement FR-01 through FR-42. Each FR should have: ID, Title, Actor, Precondition, Main
